# Система автоматизации распределения учебной нагрузки кафедры

Данный инструмент разработан для автоматизации рутинных процессов учебной части. Он решает задачу переноса, обновления и балансировки учебной нагрузки преподавателей на новый учебный год с жестким сохранением преемственности руководства магистрами.

Программа избавляет от необходимости вручную сопоставлять сотни строк таблиц формата «Осень/Весна» и сводить их воедино.


## Основная идея

Если в прошлом учебном году преподаватель руководил магистрантом 1 курса, то программа автоматически находит эту связь и в новом учебном году:
1. Закрепляет за этим же преподавателем этого студента, переводя его на 2 курс (`магистрант 2 курса`).
2. Назначает часы за руководство дипломной работой (`диплом магистра`), если студент успешно завершил обучение.

## Структура файлов проекта

* `data.py` — модуль сбора и первичной обработки данных. Считывает файлы вариантов нагрузки с помощью библиотеки pandas, парсит ФИО, дисциплины и группы, а также содержит конфигурационные списки-исключения.
* `main.py` — ядро системы. Содержит функцию tr(), которая реализует всю бизнес-логику: автоматический перенос, валидацию по спискам актуальности, точечную смену руководителей и очистку от дубликатов.

# Данные

## Вывод программы

Из-за того, что исходные CSV-файлы (`var2` и var15`) содержат огромные объемы данных по всей кафедре, функция `tr() на выходе возвращает массивный список словарей. Каждая строка вывода — это объект распределенной нагрузки в формате:

```python
{
    "t": "ФИО Преподавателя", 
    "s": "ФИО Студента / Группа / Дисциплина", 
    "k": "Тип учебной нагрузки"
}
```

### Основные списки

- `ly` — нагрузки прошлого года
- `cy` — нагрузки текущего года
- `done2` — студенты, завершившие 2 курс (для выдачи `KD`)
- `act` — актуальные студенты (если студент неактуален, его `K2`/`KD` удаляются)
- `chg` — смена руководителя: `{"Студент": "НовыйПреподаватель"}`

### Типы нагрузки

- `K1` — магистрант 1 курса
- `K2` — магистрант 2 курса
- `KD` — диплом магистра

## Как работает

1. Берутся все записи из `ly`.
2. Если у записи `k == K1`, значит это магистрант 1 курса прошлого года.
3. Для каждого студента сохраняется преподаватель из `ly` (по `K1`).
4. В `cy` для такого студента ставится `K2` с этим же преподавателем.
5. `KD` ставится только если студент есть в `done2` и в результате уже есть `K2`.
6. Если студент есть в `chg`, руководитель берется из `chg`.
7. Старые конфликтные записи `K2`/`KD` с другим преподавателем заменяются.
8. Дубли удаляются.
