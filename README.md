# Перенос преемственности магистров

## Идея

В проекте реализован перенос преемственности магистров из прошлого года в текущий.

Если в прошлом году преподаватель руководил магистрантом 1 курса, то в текущем году этому же преподавателю автоматически назначается:

- руководство магистрантом 2 курса
- диплом магистра (только если 2 курс завершен)

Новое распределение между преподавателями не выполняется.

## Файлы

- `data.py` — данные прошлого и текущего года
- `main.py` — функция переноса

## Данные

### Формат одной записи

`{"t": "...", "s": "...", "k": "..."}`

- `t` — преподаватель
- `s` — студент
- `k` — тип нагрузки

### Основные списки

- `ly` — нагрузки прошлого года
- `cy` — нагрузки текущего года
- `done2` — студенты, завершившие 2 курс (для выдачи `KD`)
- `act` — актуальные студенты (если студент неактуален, его `K2`/`KD` удаляются)
- `chg` — смена руководителя: `{"Студент": "НовыйПреподаватель"}`

### Типы нагрузки

- `K1` — магистрант 1 курса
- `K2` — магистрант 2 курса
- `KD` — диплом магистра

## Как работает

1. Берутся все записи из `ly`.
2. Если у записи `k == K1`, значит это магистрант 1 курса прошлого года.
3. Для каждого студента сохраняется преподаватель из `ly` (по `K1`).
4. В `cy` для такого студента ставится `K2` с этим же преподавателем.
5. `KD` ставится только если студент есть в `done2` и в результате уже есть `K2`.
6. Если студент есть в `chg`, руководитель берется из `chg`.
7. Старые конфликтные записи `K2`/`KD` с другим преподавателем заменяются.
8. Дубли удаляются.
